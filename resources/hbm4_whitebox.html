💡 Two FSMs run in parallel inside the HBM4 controller. The AXI FSM handles the AXI handshake protocol. The HBM CMD FSM translates requests to DRAM commands with proper timing.
AXI SLAVE FSM
IDLE
AWVALID=0, ARVALID=0
AWVALID & AWREADY → assert AWREADY
ADDR_DECODE
Latch AWADDR, AWLEN, AWID → send to CMD FSM
Assert WREADY
DATA_PHASE
Accept WDATA beats until WLAST
WLAST & WVALID & WREADY
RESP_PHASE
Assert BVALID, drive BRESP/BID
BREADY → return to IDLE
IDLE
Ready for next transaction
HBM CMD FSM (DRAM SIDE)
IDLE / INIT
PHY_INIT_DONE=0 → wait · RESET_n, CKE setup
PHY_INIT_DONE=1 & request pending
ACTIVATE
CMD=001, drive ROW_ADDR → wait tRCD cycles
tRCD elapsed
READ / WRITE
CMD=010/011, drive COL_ADDR · DQ valid · burst count
burst complete (tCL done for RD)
PRECHARGE
CMD=100 · wait tRP cycles · clear BANK_OPEN bit
tRP elapsed OR REFRESH_REQ
REFRESH
CMD=101 · all banks precharged · wait tRFC
tRFC elapsed → back to IDLE
ERROR
Bad addr / timing violation → BRESP=SLVERR · ECC_ERR_UNCORR
